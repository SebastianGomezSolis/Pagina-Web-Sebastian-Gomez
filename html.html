HTML

HTML (HyperText Markup Language) es el lenguaje de marcado que define la estructura del contenido de una página web: textos, imágenes, enlaces, tablas y formularios.

Estructura Etiquetas Formularios Semántica

1 ¿Qué es HTML?

HTML significa HyperText Markup Language. Se usa para organizar y estructurar textos, imágenes, enlaces, tablas, formularios y demás partes de una página web.

HTML no es un lenguaje de programación — es un lenguaje de marcado que describe la estructura del contenido.

2 Estructura básica de un documento HTML

HTML 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi página</title>
</head>
<body>
    <h1>Hola mundo</h1>
    <p>Mi primera página web.</p>
</body>
</html>
  • <html> — contiene todo el documento.
  • <head> — metadatos, título y enlaces a estilos.
  • <body> — contenido visible para el usuario.
  • <title> — texto que aparece en la pestaña del navegador.
  • <meta> — información sobre el documento (charset, viewport…).

3 Encabezados y párrafos

HTML 
<h1>Título principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>
<h4>Sub-sección</h4>
<h5>Nivel 5</h5>
<h6>Nivel 6</h6>

<p>Este es un párrafo normal.</p>
Solo debe existir un <h1> por página. Los demás encabezados se usan en orden jerárquico.

4 Enlaces e imágenes

HTML 
<!-- Enlace -->
<a href="https://www.google.com" target="_blank">Ir a Google</a>

<!-- Imagen -->
<img src="img/web-logo.png" alt="Logo del sitio" width="200">
Siempre incluir el atributo alt en las imágenes — mejora la accesibilidad y el SEO.

8 Etiquetas semánticas

Las etiquetas semánticas describen el propósito del contenido y mejoran la organización, accesibilidad y SEO.

HTML 
<header>Encabezado del sitio</header>
<nav>Menú de navegación</nav>
<section>
  <article>Artículo independiente</article>
</section>
<aside>Contenido relacionado</aside>
<footer>Pie de página</footer>
Usar semántica hace el código más claro para los desarrolladores, los navegadores y los lectores de pantalla (accesibilidad).

Buenas prácticas en HTML

  • Usar etiquetas correctas según el contenido (semántica).
  • Agregar alt en todas las imágenes.
  • Indentar el código para mejorar la legibilidad.
  • Separar estructura (HTML) de estilos (CSS).
  • Validar el HTML con el validador W3C.
