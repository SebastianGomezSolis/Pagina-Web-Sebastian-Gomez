Fundamentos de Internet y la Web

Esta sección explica los conceptos básicos necesarios para entender cómo funciona la comunicación en la web y cómo una página llega desde un servidor hasta el navegador.

¿Qué es Internet?

Internet es una red mundial de computadoras y dispositivos conectados entre sí. Gracias a esta red se puede intercambiar información, usar aplicaciones en línea, enviar correos y navegar por páginas web.

Ejemplo: cuando una persona ve un video en YouTube o envía un correo electrónico, está usando Internet para comunicarse con servidores remotos.

¿Qué es la World Wide Web?

La Web o World Wide Web es un servicio que funciona sobre Internet. Está formada por páginas y sitios web enlazados entre sí, los cuales se pueden visitar mediante un navegador como Chrome, Firefox o Edge.

Las páginas web se construyen principalmente con HTML, se diseñan con CSS y pueden agregar interactividad con JavaScript.

Diferencia entre Internet y Web

Analogía: Internet sería la carretera, mientras que la Web serían los sitios y páginas que se recorren por esa carretera.

Modelo OSI y sus capas

El modelo OSI es una forma de explicar la comunicación en redes mediante 7 capas. Cada una cumple una función específica y juntas permiten que los datos viajen de un origen a un destino.

1. Física
Transmite bits por cables, fibra o señales inalámbricas.
2. Enlace
Organiza tramas y controla errores locales.
3. Red
Encamina paquetes usando direcciones IP.
4. Transporte
Entrega datos completos y en orden.
5. Sesión
Administra la comunicación entre aplicaciones.
6. Presentación
Traduce, comprime y cifra la información.
7. Aplicación
Es la capa más cercana al usuario. Aquí trabajan protocolos como HTTP, FTP y SMTP.

Dirección IP

Una dirección IP identifica a un dispositivo dentro de una red. Permite saber quién envía datos y quién debe recibirlos.

  • IPv4: formato tradicional como 192.168.1.10.
  • IPv6: formato más largo y moderno con muchas más direcciones disponibles.

DNS

El sistema DNS traduce los nombres de dominio a direcciones IP. Así, una persona escribe google.com en lugar de memorizar números.

Ejemplo: el navegador pregunta al DNS cuál es la IP de un dominio y luego se conecta al servidor correcto.

Servidores, clientes y navegadores

Cliente: es el programa que hace una solicitud, como un navegador web.

Servidor: es la máquina que almacena recursos y responde a las solicitudes.

Navegador: interpreta HTML, CSS y JavaScript para mostrar la página al usuario.

Funcionamiento de una petición web

  1. El usuario escribe una URL en el navegador.
  2. El navegador consulta el DNS para encontrar la IP.
  3. Se envía una petición HTTP o HTTPS al servidor.
  4. El servidor responde con los recursos solicitados.
  5. El navegador renderiza la página.

HTTP, HTTPS, SSL y TLS

HTTP es un protocolo de comunicación usado para transferir páginas y recursos web.

HTTPS es la versión segura de HTTP. Protege la información mediante cifrado.

SSL y TLS son tecnologías de seguridad; hoy en día TLS es la más usada. Su función es cifrar la comunicación entre cliente y servidor.

Ejemplo: en una tienda en línea se usa HTTPS para proteger datos como contraseñas o información de pago.

Métodos HTTP y códigos de estado

Métodos
  • GET: obtiene información.
  • POST: envía datos nuevos.
  • PUT: actualiza un recurso.
  • DELETE: elimina un recurso.
Códigos de estado
  • 200: solicitud exitosa.
  • 404: recurso no encontrado.
  • 500: error interno del servidor.

Resumen rápido

Internet es la red, la Web es un servicio sobre esa red, y HTTP/HTTPS permiten solicitar y recibir contenido.

Concepto importante

Sin DNS sería muy difícil navegar, porque tendríamos que recordar direcciones IP numéricas para cada sitio.
