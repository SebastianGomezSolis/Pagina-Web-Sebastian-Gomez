Elementos del Backend

El backend es la parte de una aplicación que procesa datos, maneja la lógica del negocio y responde a las solicitudes del cliente. Aquí se trabaja con Spring Boot, Thymeleaf, cookies y sesiones.

Java / Spring Boot Thymeleaf Cookies & Sesiones HTTP Request/Response

1 Cliente vs. Servidor

El cliente es quien solicita un recurso, normalmente el navegador. El servidor recibe la solicitud, la procesa y envía una respuesta.

Cliente

Navegador web — envía peticiones HTTP

⇄
Servidor

Spring Boot — procesa y responde

Ejemplo: un usuario abre una página de productos; el navegador pide la información y el servidor devuelve HTML o JSON.

2 HTTP Request y HTTP Response

Request: petición que envía el cliente al servidor.

Response: respuesta del servidor con datos, estado y encabezados.

HTTP 
/* Request */
GET /usuarios HTTP/1.1
Host: localhost:8080
Accept: application/json

/* Response */
HTTP/1.1 200 OK
Content-Type: application/json

[{"id":1,"nombre":"Ana"}]

3 Métodos GET y POST

  • GET: solicita información al servidor (no modifica datos).
  • POST: envía información nueva para ser procesada o guardada.
Java 
@GetMapping("/saludo")
public String saludo() {
    return "Hola";
}

@PostMapping("/guardar")
public String guardar(@RequestBody Usuario u) {
    // guardar en base de datos
    return "Guardado";
}

4 Spring Boot

Spring Boot es un framework de Java que facilita la creación de aplicaciones web y APIs. Reduce mucha configuración manual y permite arrancar proyectos rápidamente.

Java 
@SpringBootApplication
public class Aplicacion {
    public static void main(String[] args) {
        SpringApplication.run(Aplicacion.class, args);
    }
}
Spring Boot se usa mucho porque permite desarrollar aplicaciones Java de forma más rápida y organizada que Spring tradicional.

5 Inyección de dependencias con @Autowired

La inyección de dependencias permite que Spring cree y entregue objetos automáticamente a otras clases que los necesitan, sin crear instancias manualmente.

Java 
@Service
public class UsuarioService {
    public String obtenerMensaje() {
        return "Servicio activo";
    }
}

@Controller
public class UsuarioController {
    @Autowired
    private UsuarioService usuarioService;

    @GetMapping("/mensaje")
    public String mensaje(Model m) {
        m.addAttribute("msg", usuarioService.obtenerMensaje());
        return "mensaje";
    }
}

6 @Controller y @RestController

@Controller

Devuelve el nombre de una vista (página HTML con Thymeleaf).

Java 
@Controller
public class VistaController {
    @GetMapping("/inicio")
    public String inicio() {
        return "index"; // → index.html
    }
}

@RestController

Devuelve datos directamente (JSON, texto).

Java 
@RestController
public class ApiController {
    @GetMapping("/api/mensaje")
    public String mensaje() {
        return "Hola desde la API";
    }
}

7 Uso de Thymeleaf

Thymeleaf es un motor de plantillas que permite insertar datos dinámicos dentro de archivos HTML. Se integra directamente con Spring Boot.

Java 
@Controller
public class HomeController {
    @GetMapping("/home")
    public String home(Model model) {
        model.addAttribute("mensaje", "Bienvenido al sitio");
        model.addAttribute("usuario", "Sebastián");
        return "home"; // → templates/home.html
    }
}
HTML (Thymeleaf) 
<h1 th:text="${mensaje}">Texto por defecto</h1>
<p th:text="'Hola, ' + ${usuario}"></p>

<!-- Iteración -->
<ul>
  <li th:each="item : ${lista}" th:text="${item}"></li>
</ul>

<!-- Condicional -->
<p th:if="${activo}">Sesión activa</p>

8 Cookies y Sesiones

Cookies

Pequeños datos guardados en el navegador del usuario. Sirven para recordar preferencias o mantener una autenticación.

Las cookies tienen una fecha de expiración y pueden ser seguras (HTTPS-only) o de sesión (se eliminan al cerrar el navegador).

Sesiones

Información temporal que el servidor mantiene asociada a un usuario durante su visita.

Ejemplo: en una tienda en línea, la sesión guarda el carrito de compras mientras el usuario navega por el sitio.

Flujo completo de una solicitud

  1. El usuario envía un formulario desde el navegador.
  2. El navegador hace un HTTP POST al servidor.
  3. El @Controller en Spring Boot recibe la solicitud.
  4. Un @Service inyectado procesa la lógica.
  5. La base de datos se actualiza.
  6. El servidor devuelve una respuesta (vista Thymeleaf o JSON).
