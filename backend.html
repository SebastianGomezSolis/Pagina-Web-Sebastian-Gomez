Elementos del backend

El backend es la parte de una aplicación que procesa datos, maneja la lógica del negocio y responde a las solicitudes del cliente.

Cliente vs servidor

El cliente es quien solicita un recurso, normalmente el navegador. El servidor recibe la solicitud, la procesa y envía una respuesta.

Ejemplo: un usuario abre una página de productos; el navegador pide la información y el servidor devuelve HTML o JSON.

HTTP request y HTTP response

Request: es la petición que envía el cliente al servidor.

Response: es la respuesta del servidor con datos, estado y encabezados.

Request:
GET /usuarios HTTP/1.1
Host: localhost:8080

Response:
HTTP/1.1 200 OK
Content-Type: application/json

Método GET y POST

  • GET: solicita información al servidor.
  • POST: envía información nueva para ser procesada o guardada.
@GetMapping("/saludo")
public String saludo() {
    return "Hola";
}

@PostMapping("/guardar")
public String guardar() {
    return "Guardado";
}

Spring Boot

Spring Boot es un framework de Java que facilita la creación de aplicaciones web y APIs. Ayuda a iniciar proyectos rápidamente y reduce mucha configuración manual.

@SpringBootApplication
public class Aplicacion {
    public static void main(String[] args) {
        SpringApplication.run(Aplicacion.class, args);
    }
}

Inyección de dependencias con @Autowired

La inyección de dependencias permite que Spring cree y entregue objetos automáticamente a otras clases que los necesitan.

@Service
public class UsuarioService {
    public String obtenerMensaje() {
        return "Servicio activo";
    }
}

@Controller
public class UsuarioController {
    @Autowired
    private UsuarioService usuarioService;
}

@Controller y @RestController

@Controller se usa cuando se devuelven vistas, por ejemplo páginas Thymeleaf.

@RestController se usa cuando se devuelven datos como JSON.

@Controller
public class VistaController {
    @GetMapping("/inicio")
    public String inicio() {
        return "index";
    }
}

@RestController
public class ApiController {
    @GetMapping("/api/mensaje")
    public String mensaje() {
        return "Hola desde la API";
    }
}

Uso de Thymeleaf

Thymeleaf es un motor de plantillas que permite insertar datos dinámicos dentro de archivos HTML.

@Controller
public class HomeController {
    @GetMapping("/home")
    public String home(Model model) {
        model.addAttribute("mensaje", "Bienvenido al sitio");
        return "home";
    }
}
<h1 th:text="${mensaje}">Texto por defecto</h1>

Cookies y sesiones

Cookies: pequeños datos guardados en el navegador del usuario. Sirven, por ejemplo, para recordar preferencias o mantener una autenticación.

Sesiones: información temporal que el servidor mantiene asociada a un usuario.

Ejemplo: en una tienda en línea, la sesión puede guardar el carrito de compras mientras el usuario navega por el sitio.

Resumen rápido

  • Frontend: lo que ve el usuario.
  • Backend: lógica, datos y procesamiento.
  • HTTP conecta ambas partes.

Ejemplo de flujo

  1. El usuario envía un formulario.
  2. El servidor valida los datos.
  3. La base de datos se actualiza.
  4. El servidor devuelve una respuesta.

Nota

Spring Boot se usa mucho porque permite desarrollar aplicaciones Java web de forma más rápida y organizada.
